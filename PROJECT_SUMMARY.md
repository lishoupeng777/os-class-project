# VFS 文件系统 - 项目总结报告

## 项目概述

这是一个基于 C 语言实现的虚拟文件系统，模拟 UNIX 风格的文件系统，支持多用户、多级目录、文件权限管理等核心功能。项目通过参考张尧学《操作系统教程习题与解答》中的文件系统设计理论完成。

## 核心特性

### 1. **多用户系统**
- 用户认证（用户名+密码验证）
- 用户权限隔离（UID/GID）
- 工作目录隔离（每用户独立 cwd）
- 打开文件隔离（每用户 20 个文件描述符）

### 2. **多级目录结构**
- 支持任意深度目录嵌套
- 绝对路径和相对路径解析
- 路径组件递归查找
- 目录项链接计数

### 3. **文件系统**
- UNIX 风格 Inode 结构
- 10 个块地址（8 直接 + 1 一级间接 + 1 二级间接）
- 最大文件大小 8+ MB
- 硬链接支持（链接计数）

### 4. **权限控制**
- 9 位权限模式（owner/group/other 各 rwx）
- 权限检查函数（access）
- root 用户特殊权限
- chmod 命令权限修改

### 5. **资源管理**
- **Inode 缓存**：Hash 表实现（128 个最多）
- **块分配**：空闲块栈（FAT-like）
- **Inode 分配**：空闲 Inode 栈
- **引用计数**：自动内存管理

### 6. **持久化存储**
- 虚拟磁盘文件 (virtual_disk.bin, 4 MB)
- 完整状态保存与恢复
- 延迟写策略

## 数据结构设计

### 关键数据结构

```c
// 超级块：全局文件系统元数据
struct super_block {
    int ifree[50];      // 空闲 Inode 栈
    int ffree[50];      // 空闲块栈（FAT 风格）
    // ... 管理信息
}

// Inode：文件元数据
struct dinode {
    uint16_t di_mode;   // 文件类型 + 权限
    uint8_t di_nlink;   // 硬链接计数
    uint8_t di_uid;     // 所有者 UID
    uint32_t di_size;   // 文件大小
    uint32_t di_addr[10];  // 块地址
}

// 内存 Inode：缓存
struct minode {
    dinode dino;        // 磁盘 inode 副本
    int ref_count;      // 引用计数
    int m_flag;         // 修改标志
    minode *next, *prev;  // Hash 链表
}

// 用户：多用户支持
struct user {
    char u_name[14];    // 用户名
    uint8_t u_uid;      // 用户 ID
    int u_ofile[20];    // 打开文件表
    minode *u_cwd;      // 当前工作目录
}
```

## 关键算法

### 1. 块分配 (balloc)
- **时间复杂度**：O(1) 平均
- **空间复杂度**：O(50) 栈大小
- **算法**：空闲块栈 + 链接块

### 2. 路径解析 (namei)
- **时间复杂度**：O(深度 × 目录大小)
- **算法**：逐级查找目录项 → Inode

### 3. Inode 缓存 (iget/iput)
- **查找**：Hash 表 O(1) 平均
- **管理**：引用计数自动释放
- **最多缓存**：128 个 Inode

### 4. 权限检查 (access)
- **检查模型**：Owner/Group/Other
- **特殊用户**：root (UID=0) 无限权限

## 技术亮点

### 内存管理
```c
// 引用计数实现
minode *mip = iget(ino);    // ref_count++
iput(mip);                  // ref_count--
// 当 ref_count==0 自动释放
```

### 权限模型
```c
// UNIX 9 位权限
mode = S_IFDIR | 0755
     = 0x4000 | 0755
     = 目录，权限 755 (rwxr-xr-x)
```

### 路径解析
```c
// 支持相对和绝对路径
namei("/a/b/c")     // 绝对路径
namei("d/e")        // 相对路径
namei("../f")       // 上级目录
```
