# 📋 VFS 虚拟文件系统 — 验收演示流程

> 演示环境：Windows 终端 (CMD)  
> 默认管理员账号：`root` / `root`

---

## 第一步：启动准备

```bash
# 删除旧磁盘文件（确保全新启动）
del virtual_disk.bin

# 编译
gcc -Wall -Wextra -g -Iinclude src\core\main.c src\core\vfs_globals.c src\core\vfs_disk.c src\core\vfs_block.c src\core\vfs_inode.c src\core\vfs_path.c src\core\vfs_perm.c src\commands\commands_system.c src\commands\commands_dir.c src\commands\commands_file_io.c src\commands\commands_file_ops.c src\commands\commands_user.c -o bin\vfs_new.exe

# 运行
bin\vfs_new.exe
```

## 第二步：用户管理（功能 1-4）

### 1️⃣ 登录系统
```
VFS> login
Username: root
Password: root
Login successful!
```

### 2️⃣ 查看当前用户
```
VFS> whoami
uid=0(root) gid=0
```
> **📌 解释**：显示当前用户 UID=0（root 用户），GID=0（root 组）

### 3️⃣ 添加新用户
```
VFS> useradd alice
Password: 123456
User added successfully!
```

### 4️⃣ 切换用户
```
VFS> logout
Logout successful!

VFS> login
Username: alice
Password: 123456
Login successful!

VFS> whoami
uid=1(alice) gid=1
```
> **📌 解释**：普通用户有自己的 UID/GID，与 root 隔离

### 5️⃣ 权限控制 — 普通用户无法删除用户
```
VFS> useradd bob
Permission denied! Only root can add users!
```
> **📌 解释**：普通用户无权执行管理操作

### 6️⃣ root 改密码
```
VFS> logout
VFS> login
Username: root
Password: root
Login successful!

VFS> passwd alice
New password: abc123
Confirm new password: abc123
Password changed successfully!
```

## 第五步：文件打开/读写操作（功能 16-19）

### 1️⃣6️⃣ 打开文件
```
VFS> open hello.txt
fd = 0
```
> **📌 解释**：返回文件描述符 fd=0，类似 UNIX 的 open() 系统调用

### 1️⃣7️⃣ 读取文件内容
```
VFS> read 0 100
Hello, this is a test file!
This is the second line.
```
> **📌 解释**：从 fd 0 读取最多 100 字节

### 1️⃣8️⃣ 写入文件
```
VFS> write 0
Enter text (end with Ctrl+Z or Enter an empty line):
This is new content!
^Z

VFS> cat hello.txt
This is new content!
```
> **📌 解释**：写入覆盖了原内容

### 1️⃣9️⃣ 关闭文件
```
VFS> close 0
```
> **📌 解释**：释放文件描述符，类似 UNIX 的 close()

## 第六步：权限管理（功能 20-21）

### 2️⃣0️⃣ 修改文件权限
```
VFS> chmod 644 hello.txt
Permissions changed successfully!
VFS> stat hello.txt
Mode: 0x81A4
Size: 19 bytes
UID: 0
```
> **📌 解释**：644 = -rw-r--r--（所有者读写，其他人只读）

### 2️⃣1️⃣ 修改文件所有者（仅 root）
```
VFS> chown 1 hello.txt
Ownership changed successfully!
```
> **📌 解释**：将文件所有者改为 UID=1（alice 用户）

```

```
> **📌 解释**：硬链接指向同一 inode，内容完全相同

> **📌 解释**：符号链接是一个特殊文件，指向目标路径

## 权限模型说明（重要改进）

以前：所有用户共享根目录，普通用户被拒绝写权限  
现在：**每个用户拥有独立的 home 目录**

- `root` — 在 `/` 根目录工作，root 拥有所有权限
- 新用户 `alice` — 自动创建 `/home/alice` 目录，**在该目录下拥有完整读写权限**
- 用户无法在别人的 home 里写文件（权限隔离）
- root 可以访问和修改任何位置

## 老师可能的提问及回答

**Q: 虚拟磁盘是怎么实现的？**
> 程序在内存中分配 4MB 缓冲区，划分为超级块、Inode 表、数据块三部分。用户所有操作都在内存中进行，退出时 `save_to_disk()` 将整个缓冲区写入 `virtual_disk.bin` 文件，启动时 `load_from_disk()` 读回内存。

**Q: 用户数据怎么保存的？**
> 虚拟磁盘 4MB 的最后 512 字节预留用来存储用户表（用户名、密码、UID、GID、工作目录 inode 号）。每次 `save_to_disk()` 时序列化写入，`load_from_disk()` 时反序列化恢复。

**Q: 权限模型是什么？**
> 三级权限：所有者(owner)→同组用户(group)→其他用户(other)。root(UID=0) 可绕过所有权限检查。

**Q: 文件怎么寻址的？**
> Inode 包含 10 个地址项：前 8 个直接块指针、1 个一级间接指针、1 个二级间接指针。支持最大文件 8×512 + 128×512 + 128×128×512 ≈ 8.5MB。
