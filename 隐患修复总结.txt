【隐患 #2: Inode 缓存泄漏修复】

在 vfs.c 中添加：

1. 强化 iput() 函数
   ├─ 检查 ref_count <= 0 的异常情况
   ├─ 输出警告信息便于调试
   └─ 防止多次释放同一 inode

2. 新增 check_inode_cache_health() 函数
   ├─ 遍历所有缓存中的 inode
   ├─ 检测引用计数异常 (ref_count > 5)
   └─ 打印诊断信息

【隐患 #3: 符号链接循环检测修复】

在 vfs.c namei() 函数中添加：

1. 定义深度限制常量
   #define MAX_SYMLINK_DEPTH 10

2. 在路径解析循环中检查
   if (symlink_depth > MAX_SYMLINK_DEPTH) {
       // 快速失败，防止无限循环
   }

【隐患 #4: 寻址越界检查修复】

在 vfs.c balloc() 函数中添加：

1. 块号范围验证
   if (blkno < DATASTART/BLOCKSIZ || blkno >= DATASTART/BLOCKSIZ+FILEBLK)
       return -1;  // 拒绝无效块号

【隐患 #5: 并发删除检查修复】

在 commands.c cmd_delete() 函数中添加：

1. 检查打开文件表
   for (int i = 0; i < SYSOPENFILE; i++) {
       if (sys_ofile[i].of_inode_num == mip->ino && 
           sys_ofile[i].of_count > 0) {
           file_in_use = 1;
           break;
       }
   }

2. 拒绝删除已打开的文件
   if (file_in_use) {
       printf("Error: Cannot delete - currently open\n");
       return;
   }
